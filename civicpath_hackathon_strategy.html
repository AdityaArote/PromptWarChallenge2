5 User Stories

Story 1 — First-time voter
As a first-time voter, I want a guided, step-by-step visual walkthrough of the entire election process — from registration to result — so that I never feel confused or miss a critical deadline.
CRITICAL STORY — Most users are exactly this persona. Solving this wins the demo.
Story 2 — Civic educator
As a civic educator, I want to share an embeddable, shareable link to specific election phases so that I can use CivicPath as a classroom tool without any setup.
High Impact
Story 3 — NRI / overseas voter
As an overseas citizen, I want to ask a conversational AI assistant specific questions about absentee/postal voting timelines so that I don't lose my vote to a missed deadline.
AI differentiation
Story 4 — Youth voter
As a young voter, I want a gamified quiz about election civics with instant explanations so that learning feels engaging and I can track my progress over time.
Engagement + retention
Story 5 — Accessibility-first user
As a user with low digital literacy or a visual impairment, I want the entire experience to work via keyboard navigation and screen reader so that CivicPath is truly inclusive.
Evaluation criterion: Accessibility

Most Critical Story → Story 1 (First-time voter journey)

This is the core demo story. If a judge sees a confused voter land on the app and leave knowing every step with a personalized timeline — that's the win. Everything else is additive. Build this first, deeply, before touching anything else.

Stitch

TypeScript = Code Quality. Vitest = Testing. react-aria = Accessibility. One framework, all four criteria covered.

Winning architectural pattern: Edge-first, AI-augmented

All AI calls go through a Supabase Edge Function — never directly from the client. This means: (1) API key never exposed = Security points, (2) Response caching per question = Efficiency points, (3) Rate limiting per user = Security + Efficiency. One pattern, four criteria addressed.

2-Hour Hackathon Sprint

0:00
Scaffold + Seed (0–15 min)
  • npm create vite@latest civicpath -- --template react-ts
  • Install: tailwind, zustand, react-router, @supabase/supabase-js, @anthropic-ai/sdk
  • Run supabase init + create project + write migration SQL
  • Add 3 env vars: VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY, GEMINI_API_KEY
  • Stitch: Run ingest script to seed election phase data into Supabase
0:15
Core Data + Auth (15–30 min)
  • Supabase schema: election_phases, quiz_questions, user_progress, deadlines
  • Enable Supabase anonymous auth (no friction for judges)
  • Enable RLS policies — SELECT public, INSERT authenticated only
  • Write useElectionData hook — fetches phases sorted by order
  • Deploy Supabase Edge Function: gemini-chat proxy with rate limiting
0:30
Critical Story: Journey Page (30–75 min) 🔥
  • Build InteractiveTimeline component — vertical stepper, each phase clickable
  • PhaseCard: title, description, deadline badge, "Mark complete" button
  • Wire Antigravity: entrance animation per card, completion animation
  • Floating AI chat bubble — opens ChatWindow, auto-prefilled with current phase context
  • useGeminiChat hook: streams Gemini response through Edge Function
  • Voter type selector on Home (first-time / returning / overseas) → personalizes journey
  • Persist completed steps to Supabase user_progress table
1:15
Quiz Module + Polish (75–105 min)
  • QuizCard component: question, 4 options, instant AI explanation on wrong answer
  • Score saved to Supabase, streak counter in UI header
  • Write 3 Vitest unit tests: quiz scoring logic, deadline formatter, phase sort
  • Accessibility pass: aria-labels, keyboard nav, focus rings, color contrast check
  • Add lang switcher → calls Gemini for on-the-fly translation
1:45
Deploy + README (105–120 min)
  • docker build → push to GCP Artifact Registry → deploy Cloud Run
  • Set Cloud Run env vars, verify HTTPS endpoint
  • Add GCP Cloud Logging — log every Gemini call (satisfies Google Services criterion)
  • Write README: architecture diagram, setup steps, tech stack rationale
  • Final smoke test: full voter journey, quiz, AI chat, keyboard-only nav
  • Submit with live URL + GitHub link

Evaluation Criteria — Your Edge

Code Quality

  • TypeScript strict mode throughout
  • Custom hooks isolate all side effects
  • No inline styles — Tailwind only
  • Consistent error boundaries
  • README explains every decision

Security

  • Gemini key only in Edge Function (server-side)
  • Supabase RLS on all tables
  • Input sanitization before Gemini prompt
  • Anonymous auth — no PII collected
  • HTTPS enforced on Cloud Run

Efficiency

  • Gemini response cache in Edge Function (KV store)
  • Vite code splitting per route
  • Supabase select only needed columns
  • Lazy-load quiz module
  • Stitch pre-seeds data — no runtime fetch

Testing

  • Vitest: quiz scoring logic
  • Vitest: deadline date formatter
  • Vitest: phase sort order
  • Mention test coverage in README
  • Add test:ci script to package.json

Accessibility

  • react-aria for all interactive elements
  • Full keyboard navigation on timeline
  • ARIA live regions for AI chat
  • 4.5:1 color contrast minimum
  • Skip-to-content link in header

Google Services

  • Vertex AI (Gemini 1.5 Flash) — AI chat
  • Cloud Run — app deployment
  • Cloud Logging — audit trail
  • Cloud Build — CI/CD pipeline
  • Artifact Registry — Docker images

3 Things That Put You in the Top 1%

01

The AI knows where the user is

When the judge opens the chat on the "Candidate Filing" phase, Gemini already has that context. It doesn't ask "what do you want to know?" — it says "You're on candidate filing. Deadlines typically fall 45 days before election day. What would you like clarified?" That's not a chatbot. That's a civic companion.

02

Antigravity makes the timeline feel inevitable

Every phase card slides in with a stagger. Completing a step triggers a satisfying completion pulse. The progress bar animates smoothly. Judges have seen 40 React apps today. Yours is the only one that feels like it was designed, not built.

03

Security is invisible but verifiable

Your Supabase Edge Function logs every request. Your RLS policies are visible in the Supabase dashboard. Your API key is not in the client bundle. When the AI evaluator scans your code, it finds zero exposed secrets and clean separation of concerns. That's a perfect Security score.
